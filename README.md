# RAG Pipeline — Вопрос-ответ по PDF-документам

Jupyter-ноутбук для Google Colab, реализующий RAG (Retrieval-Augmented Generation) — систему, которая ищет релевантные фрагменты в загруженных PDF-файлах и генерирует ответ с помощью GPT.

## Как это работает

**Этап 1 — Индексация** (выполняется один раз):

1. PDF-файлы загружаются и извлекается текст
2. Текст делится на перекрывающиеся фрагменты (chunks)
3. Каждый фрагмент переводится в векторное представление (эмбеддинг)
4. Векторы сохраняются в локальную ChromaDB

**Этап 2 — Поиск и генерация** (при каждом вопросе):

1. Вопрос пользователя переводится в вектор
2. В ChromaDB находятся Top-K наиболее похожих фрагментов
3. Фрагменты + вопрос отправляются в GPT
4. GPT генерирует ответ строго на основе найденных фрагментов

## Требования

- Google Colab
- API-ключ OpenAI ([получить](https://platform.openai.com/))

## Быстрый старт

**1. Откройте ноутбук в Google Colab**

**2. Добавьте API-ключ в Secrets Colab**

Панель Secrets (иконка 🔑 слева):
- Название: `openai_api_key`
- Значение: ваш ключ OpenAI

**3. Загрузите PDF-файлы**

Создайте папку `Файлы_pdf` в панели Файлы и добавьте туда PDF (минимум 2 файла), либо загрузите zip-архив и распакуйте его (Пункт Загрузка PDF-файлов).

**4. Запустите все ячейки по порядку**

В последней ячейке измените `user_input` на свой вопрос.

## Настройки

В файле `config.json` (создаётся автоматически):

| Параметр | По умолчанию | Описание |
|---|---|---|
| `llm_model` | `gpt-4o-mini` | Модель GPT |
| `base_url` | `https://api.openai.com/v1` | Базовый URL API |
| `timeout_sec` | `60` | Таймаут запроса |

В коде разбиения на чанки:

| Параметр | По умолчанию | Описание |
|---|---|---|
| `chunk_size` | `1000` | Размер фрагмента в символах |
| `chunk_overlap` | `200` | Перекрытие между фрагментами (~20%) |
| `k` | `5` | Количество фрагментов для поиска (Top-K) |
